# AI聊天总结

在当前群聊或私聊中发送 `/ai 总结`，插件会读取当前会话最近一段聊天记录，调用你配置的大模型接口生成简短总结，并直接发送到当前聊天。

## 使用命令

```text
/ai 总结
```

按配置里的默认条数总结当前会话最近聊天记录。开始执行和失败时会用浮动提示反馈状态，成功后直接发送总结到当前聊天。

```text
/ai 总结 120
```

临时总结最近 120 条聊天记录。可选范围为 50~200，超出范围会自动限制到边界值。

```text
/ai 配置
```

打开配置窗口，设置 API 地址、API Key、模型名称、默认总结条数、总结提示词和运行日志开关。

```text
/ai 帮助
```

弹窗显示使用说明。

## 配置说明

配置通过插件设置窗口保存到 `config.prop`：

- `api_url`：大模型接口地址，默认是 OpenAI 风格的 `/v1/chat/completions` 地址。
- `api_key`：接口密钥。插件会自动补 `Bearer ` 前缀，也可以手动填写完整 `Bearer xxx`。
- `model`：模型名称。
- `summary_count`：默认读取最近多少条消息，范围 50~200。
- `log_enable`：是否开启运行日志，默认 `false`。关闭后插件不会写入 WAuxiliary 日志，避免日志文件过大；排查问题时可在配置窗口开启。
- `summary_prompt`：总结用系统提示词。

## 工作方式

插件使用 WAuxiliary 提供的 `queryHistoryMsg(...)` 读取当前聊天历史，整理文本、图片、语音、文件、引用等消息的简短表示，再按 `messages` 格式提交给模型接口。默认适配 OpenAI 兼容的 `/v1/chat/completions` 返回结构；如果 API 地址是 Anthropic Claude 原生 `/v1/messages`，会自动改用 `x-api-key` 和 `anthropic-version` 请求头。返回内容会以 `【AI聊天总结】` 开头直接发送到当前聊天。

## 调试

接口调用失败或聊天记录查询异常时，插件仅在 `log_enable = true` 时写入 WAuxiliary 日志。日志可能包含调用时间、API 地址、脱敏后的 API Key、模型名称、错误信息以及聊天记录查询过程。默认关闭，排查问题时可在 `/ai 配置` 中开启。

## 注意事项

- 聊天记录会发送到你配置的外部 API，请确认接口服务可信。
- 执行总结过程不会在聊天窗口发送中间消息，只会用浮动提示显示开始或失败状态，成功后直接输出结果。
- 插件运行在手机端 Android 环境，无法在仓库内直接完整测试，请在 WAuxiliary 中实际加载测试。
